# SDD — Spec-Driven Development

This project uses a local-first SDD workflow.

Every relevant change should move through:

```text
idea -> PRD -> SPEC -> tasks -> implementation -> archive
```

## Start Here

- [Constitution](./explanation/constitution.md) — project rules for humans and AI agents.
- [Changelog](./CHANGELOG.md) — delivered changes.
- `docs/changes/` — active and delivered changes.
- `docs/adr/` — architectural decisions.
